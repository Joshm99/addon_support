Tag Tool

A minimal entity tagging utility for Bedrock. Add, remove, and inspect tags on any entity through a clean in-game Script API UI — no commands needed.

Version 1.0.0
Namespace jm_tt:
Bedrock 1.21+
Script API 2.x

Overview

Tag Tool is a lightweight addon for server admins and addon developers who need to inspect or modify entity tags in-game without using the /tag command. Everything happens through a multi-screen Script API UI — just interact with an entity and you're in.

It's intentionally small. No economy system, no player registry, no database. Just a focused tool for tag management.

Installation

  1. Download the .mcpack file from CurseForge or the release page.
  2. Open it — Minecraft will automatically import both the behavior pack and resource pack.
  3. Create or open a world and go to Behavior Packs in world settings.
  4. Activate JM Tag Tool BP. The resource pack will be added automatically.
  5. Enable Beta APIs under Experiments in world settings.
ℹ
Note

Tag Tool requires the Beta APIs experiment enabled. This is standard for all Script API addons on Bedrock 1.21+.

Usage

Opening the UI

Sneak + interact (right-click / use button) on any entity to open the Tag Tool interface. The main screen shows all tags currently on that entity.

Adding a tag

From the main screen, select Add Tag. Type the tag name into the text field and confirm. The tag is applied immediately — no reload needed.

Removing a tag

From the main screen, select any existing tag from the list. You'll get a confirmation screen before the tag is removed.

★
Tip

Tag Tool works on all entity types — mobs, players, armor stands, item frames, and more. If it's an entity, you can tag it.

Commands

Tag Tool is UI-first and doesn't add custom slash commands. Standard Bedrock /tag commands still work alongside it.

# Native Bedrock tag commands still work as expected
/tag @e[type=cow] add my_custom_tag
/tag @e list

Compatibility

Tag Tool uses a minimal footprint to avoid conflicts with other packs:

  • No custom blocks or items that could conflict with other packs
  • No global scoreboard entries
  • No dynamic property namespace collisions — uses jm_tt: prefix throughout
  • Achievement-safe — does not add any blocks or items
★
Compatibility

Works alongside AdminTools, JM Shop, JM Mail Delivery, and JM Notice Board on the same world without conflicts.
